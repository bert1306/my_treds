# Мастер: зависимость шагов и опций от шаблона и роли

## Текущее состояние

- **5 шагов:** goal → goalDetail → role → detailLevel → context (последний опциональный).
- При входе по **пресету** (кнопка): goal и goalDetail уже заданы → по факту пользователь видит шаги 3, 4, 5 (роль, детализация, контекст).
- Если в **профиле** задана роль: при GET/POST мастера она подставляется в `collected`, шаг «роль» не показывается.

## Что нужно по шаблону (пресету)

| Поле        | trends | formulate | ideas | summary | Комментарий |
|------------|--------|-----------|--------|---------|-------------|
| goal       | ✓ preset | ✓ preset | ✓ preset | ✓ preset | Не спрашиваем при пресете |
| goalDetail | ✓ preset | ✓ preset | ✓ preset | ✓ preset | Не спрашиваем при пресете |
| role       | опц.   | опц.     | опц.   | опц.   | Есть в шаблоне `{{#role}}`; если есть в профиле — не спрашиваем |
| detailLevel| да     | да       | да     | да     | Всегда нужен для формата ответа |
| context    | да     | да       | да     | **важно** | Для summary — исходный текст; для остальных — тема/вопрос |

Итог для **режима пресета**: спрашиваем только **роль** (если нет в профиле), **detailLevel**, **context**. То есть 1–3 шага вместо 5.

## Сокращение опций по шаблону

### detailLevel

- **trends, formulate, ideas:** все 4 варианта уместны (краткий обзор, подробно, пошагово, сравнение).
- **summary:** по смыслу «уровень сжатия» — достаточно **краткий обзор** и **подробное объяснение** (структурированное резюме). «Пошагово» и «сравнение» можно скрыть, чтобы не раздувать выбор.

### role

- Варианты роли общие для всех шаблонов; сужение по шаблону (например, для trends только analyst, marketer) можно ввести позже. Пока оставляем полный список + «Не указывать».

## Реализация

1. **Режим пресета:** если в `collected` уже есть пара (goal, goalDetail), соответствующая одному из пресетов, считаем, что мы в «пресетном» потоке: следующие шаги — только role (если нет в профиле), detailLevel, context.
2. **Нумерация шагов:** в пресетном потоке отображать «Шаг 1 из 3», «Шаг 2 из 3», «Шаг 3 из 3» (относительная нумерация оставшихся шагов).
3. **Опции detailLevel для summary:** для пресета `summary` показывать только [Краткий обзор, Подробное объяснение]; для остальных пресетов — все 4.
4. **Роль из профиля:** уже учитывается в API (подстановка в `collected`) — шаг «роль» не показывается; менять не требуется.

## Итоговая логика

- **Вход «Новый диалог» (без пресета):** все 5 шагов, полные опции.
- **Вход по кнопке пресета:** 1–3 шага (роль при отсутствии в профиле → detailLevel → context), нумерация «1 из N».
- **Пресет summary:** 2 варианта детализации (brief, detailed); остальные пресеты — 4 варианта.
