# Мастер: шаги и опции в зависимости от шаблона и роли

## Принцип

Цель и уточнение (goal, goalDetail) при входе по **пресету** уже заданы кнопкой → не спрашиваем. Спрашиваем только то, что реально нужно для выбранного сценария; варианты выбора подбираем под шаблон, без лишних опций.

---

## Анализ по шаблонам (пресетам)

### 1. Тренды 2026 (preset: trends)

**Суть запроса:** «Опиши ключевые тренды в области X» — сфера из контекста, объём ответа (кратко или подробно).

**Что нужно от пользователя:**
- **Роль** — опционально, для адаптации примеров (если есть в профиле — не спрашиваем).
- **Детализация** — по смыслу это «объём/формат описания трендов». Уместны только:
  - **Краткий обзор** — 5–7 тезисов.
  - **Подробное объяснение** — развёрнуто с примерами.
- Варианты «Пошаговая инструкция» и «Сравнение вариантов» для трендов избыточны: запрос не про план действий и не про сравнение направлений в первую очередь. Оставляем **2 варианта**.

**Контекст:** тема/сфера (например: «IT», «маркетинг», «образование»). Опционально можно пропустить.

**Итог по шагам:** роль (если нет в профиле) → детализация (2 варианта) → контекст.

---

### 2. Помоги сформулировать задачу (preset: formulate)

**Суть запроса:** из размытой цели получить один готовый запрос к ИИ — с указанием формата ответа (кратко, подробно, план, сравнение).

**Что нужно:**
- **Роль** — опционально.
- **Детализация** — здесь это «какой формат ответа нужен у результата». Все **4 варианта** уместны: краткий, подробный, пошаговый план, сравнение.
- **Контекст** — цель или вопрос пользователя (обязательно по смыслу, но технически можно пропустить).

**Итог по шагам:** роль → детализация (4 варианта) → контекст.

---

### 3. Идеи для проекта (preset: ideas)

**Суть запроса:** идеи по теме/проекту — сколько и насколько развёрнуто.

**Что нужно:**
- **Роль** — опционально.
- **Детализация** — «сколько идей и в каком виде»: кратко (тезисы), подробно (с пояснением), пошагово (идеи + первые шаги). Вариант «Сравнение вариантов» для генерации идей менее типичен — можно не показывать. Оставляем **3 варианта**: краткий обзор, подробное объяснение, пошаговая инструкция.
- **Контекст** — тема или проект.

**Итог по шагам:** роль → детализация (3 варианта) → контекст.

---

### 4. Краткое резюме (preset: summary)

**Суть запроса:** сократить/резюмировать текст; уровень сжатия.

**Что нужно:**
- **Роль** — опционально.
- **Детализация** — по смыслу «уровень сжатия»: краткий обзор (5–7 тезисов) или структурированное резюме. Достаточно **2 варианта**: краткий обзор, подробное объяснение.
- **Контекст** — исходный текст или описание (критично).

**Итог по шагам:** роль → детализация (2 варианта) → контекст.

**Роль:** если задана в профиле — шаг «роль» не показываем. Варианты роли общие для всех шаблонов (полный список + «Не указывать»).

**Вход «Новый диалог» (без пресета):** все 5 шагов (goal → goalDetail → role → detailLevel → context), для detailLevel — все 4 варианта.

---

## Реализация в коде

- Для каждого пресета задан свой массив опций `detailLevel` (2, 3 или 4 варианта).
- В пресет-режиме при шаге «детализация» подставляются опции по `presetId`.
- Нумерация «Шаг X из Y» по-прежнему считается от оставшихся шагов (роль при отсутствии в профиле, детализация, контекст).
