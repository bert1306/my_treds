# Площадка под новый проект (один сервер)

Сервер один — Beget VPS. Старое приложение (my_treds) снято с деплоя. Ниже: что уже есть на сервере и как развернуть **новый** проект на этой же площадке.

## Что уже есть на сервере

После очистки на сервере (root@5.35.89.34) есть:

| Компонент | Состояние |
|-----------|-----------|
| **Node.js 20** | Установлен (`node -v`) |
| **npm** | Есть |
| **Nginx** | Установлен, слушает 80 порт |
| **PM2** | Установлен глобально, процессов нет (список пустой) |
| **Каталог /root/my_treds** | Удалён |
| **Ollama** | Не запущена (при необходимости можно поставить заново) |

Nginx, скорее всего, по-прежнему настроен на проксирование на `127.0.0.1:3000`. Пока на 3000 ничего не крутится — сайт по IP/домену будет отдавать ошибку (502 или connection refused). Это нормально до деплоя нового приложения.

## Подготовка площадки под новый проект

### Вариант A: новый проект займёт порт 3000 (как раньше)

Тогда Nginx трогать не нужно — достаточно развернуть приложение и запустить его на 3000.

1. **Создать каталог и клонировать новый репозиторий**
   ```bash
   ssh -i ~/.ssh/id_ed25519_beget root@5.35.89.34
   cd /root
   git clone https://github.com/ВАШ_ЛОГИН/НОВЫЙ_РЕПО.git
   cd НОВЫЙ_РЕПО
   ```

2. **Установка зависимостей и сборка** (для Next.js — ориентир)
   ```bash
   cp .env.example .env   # если есть
   nano .env              # заполнить переменные
   npm ci
   # если есть Prisma:
   npx prisma migrate deploy
   npm run build
   ```

3. **Запуск под PM2 под новым именем**
   - В репозитории нового проекта добавьте свой `ecosystem.config.cjs` (или скопируйте и переименуйте `name` и `cwd`).
   - Либо одной командой:
   ```bash
   pm2 start node_modules/.bin/next --name "new_app" -- start
   # или для другого типа приложения — свой script
   pm2 save
   pm2 startup   # если ещё не делали — выполнить выведенную команду
   ```

4. **Проверка**  
   Открыть в браузере `http://5.35.89.34` (или ваш домен). Nginx уже проксирует на 3000.

### Вариант B: новый проект на другом порту (например 3001)

Если приложение должно слушать, например, 3001:

1. Развернуть проект в `/root/НОВЫЙ_РЕПО`, собрать, запустить через PM2 с переменной `PORT=3001` (или как принято в проекте).
2. **Поправить Nginx**, чтобы проксировать на 3001:
   ```bash
   sudo nano /etc/nginx/sites-available/my_treds
   # заменить proxy_pass http://127.0.0.1:3000 на http://127.0.0.1:3001
   sudo nginx -t && sudo systemctl reload nginx
   ```

### Что при необходимости обновить в Nginx

- Файл конфига мог называться `my_treds` — его можно отредактировать или заменить на новый (например `new_app`), затем в `sites-enabled` сделать симлинк на новый конфиг и убрать старый.
- Таймауты (например для чата/Ollama) оставлены в текущем конфиге; для нового проекта без долгих запросов можно уменьшить.

## Краткий чеклист перед деплоем нового проекта

- [ ] Репозиторий нового проекта доступен (GitHub и т.д.)
- [ ] На сервере: Node 20, Nginx, PM2 установлены (уже есть)
- [ ] Решено: приложение на порту 3000 или другом (и при другом — обновлён Nginx)
- [ ] В новом проекте есть свой `.env.example` и при необходимости свой `ecosystem.config.cjs` (или аналог для PM2)

После деплоя нового приложения площадка будет обслуживать только его; старый my_treds на этом сервере больше не используется.
