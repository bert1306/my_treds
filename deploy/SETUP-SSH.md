# Вход по ключу под root — с нуля

Все команды по порядку. Сервер: **root@5.35.89.34**, проект будет в **/root/my_treds**.

---

## Шаг 0. Пароль root на Beget

1. Зайдите в панель Beget → **Серверы** → ваш VPS.
2. Найдите раздел **«Доступ» / «SSH» / «Управление доступом»**.
3. **Установите или сбросьте пароль для root** (кнопка «Задать пароль» / «Сбросить пароль»).
4. Запомните пароль — он понадобится один раз для `ssh-copy-id`.

Без пароля root вход по ключу настроить нельзя (нужен один вход с паролем, чтобы скопировать ключ).

---

## Шаг 1. Зачистка на Mac (старый ключ и кэш для этого сервера)

Выполните **в терминале на Mac**:

```bash
# Удалить старый ключ хоста для этого IP (чтобы не было конфликтов)
ssh-keygen -R 5.35.89.34
```

(Предупреждение «not found» — нормально, значит записи не было.)

---

## Шаг 2. Новый SSH-ключ на Mac

Сгенерировать **новый** ключ только для этого сервера (старый можно не трогать):

```bash
ssh-keygen -t ed25519 -C "mac-beget-root" -f ~/.ssh/id_ed25519_beget -N ""
```

`-N ""` — без пароля на ключ (два раза Enter, если спросит). Проверка:

```bash
ls -la ~/.ssh/id_ed25519_beget.pub
```

---

## Шаг 3. Скопировать ключ на сервер (под root)

Будет запрошен **пароль root** из шага 0:

```bash
ssh-copy-id -i ~/.ssh/id_ed25519_beget.pub root@5.35.89.34
```

Введите пароль root. В конце должно быть: «Number of key(s) added: 1».

Если ошибка «Connection closed» — проверьте, что в панели Beget действительно задан пароль root и что вход по паролю для root разрешён (в описании VPS/документации Beget).

---

## Шаг 4. Проверка входа без пароля

Подключение с новым ключом (явно указать ключ):

```bash
ssh -i ~/.ssh/id_ed25519_beget root@5.35.89.34 "echo ok"
```

Должно вывести `ok` без запроса пароля.

Чтобы не указывать ключ каждый раз, добавьте в `~/.ssh/config` на Mac:

```
Host beget-my_treds
    HostName 5.35.89.34
    User root
    IdentityFile ~/.ssh/id_ed25519_beget
```

Тогда достаточно: `ssh beget-my_treds "echo ok"`.

---

## Шаг 5. (Опционально) Убрать ключ у пользователя albert на сервере

Если больше не планируете заходить под `albert`, можно удалить его ключи на сервере. Сначала зайдите под root:

```bash
ssh -i ~/.ssh/id_ed25519_beget root@5.35.89.34
```

На сервере (если есть каталог albert):

```bash
rm -f /home/albert/.ssh/authorized_keys
# или удалить весь каталог: rm -rf /home/albert/.ssh
exit
```

Если пользователя albert нет или каталога нет — этот шаг пропустить.

---

## Готово

После успешного шага 4 напишите в чат: **«Готово, ключ под root настроен»** — дальше можно будет выполнять команды на сервере и делать деплой в `/root/my_treds`.
